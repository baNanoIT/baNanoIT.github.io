Task Project

Es un proyecto que tiene como objetivo crear un sistema de administración de tareas, permitiendo al usuario tener una mejor organización de sus actividades.

Ofrece funcionalidades de guardado y acceso rápido a la información más importate de la misms como lo es:


  • Nombre
  • Descripción
  • Fecha de creación
  • Estado de la tarea

Base de datos

TaskProject hace uso del formato JSON para poder preservar la información de cada usuario y a su vez tener disponibilidad 24/7 alrededor del globo. Esta información es almacenada en la nube, específicamente Firebase de Google y por otro lado un entrono local MongoDBCompass de MongoDB.

Cabe resaltar que Flare Dev. Team está en la quiebra por lo que el servicio prestado através de la nube es gratuito en ambas plataformas (alivianenos con una feria por favor).


MongoDB Atlas

Para realizar la conexión a la base de datos previamente se debió de haber creado un cluster y una colección. Después de eso obtenemos la dirección URL de acceso a la base de datos que es la información que se manda como paramétro "MongoURL".

Firebase

Para realizar la conexión a la base de datos previamente se debió de haber creado dicha base de datos en la plataforma Firebase. Firebase facilita la descarga de un archivo .JSON que contiene toda la información para establecer la conexión en el código dicha información va encapsulada en la variable "serviceAccount". 



  Método para realizar la conexión con Firebase

  admin.initializeApp({
      credential: admin.credential.cert(serviceAccount)
    });

Endpoints

Los Endpoints son la convergencia de una ruta, permite ejecutar código específico para responder a la solicitud cualquier petición HTTP que llegue al servidor.

Esto quiere decir que las peticiones GET, POST, PUT y DELETE poseen su propio Endpoint para operar.

Obtener Información

Las solicitudes GET son aquellas que el usuarios sin saberlo manda al servidor para obtener información específica. En este proyecto dicha solicitud puede ir por distintos caminos.

La ruta Get All Tasks se ejecuta al momento de querer acceder a toda la información, consultando todas las tareas registrados en la base de datos. 

 Get All Tasks | localhost:3000/api/mongodb/task/ 

 Get All Tasks | localhost:3000/api/firebase/task/

Crear Tarea

Las solicitudes POST son las que se utilizan para poder crear nueva información dentro de una base de datos. En este caso crear una nueva tarea.
Por ejemplo supongamos que tenemos una archivo JSON con las siguiente propiedades:
"title": "TareaPost", "body": "TareaPostUsuarioAdmin", "creation_date": "28/10/2024, 20:35:28", "completed": false.

Podemos probar como reaccionaría el sistema al momento de generar un post sobre este JSON (estamos con un usuario llamado admin). 

 Post | localhost:3000/api/mongodb/task/


Resultado |

Actualizar proyecto

Las solicitudes PUT nos ayudan a actualizar documento de la base de datos.

Nota: Sobreescribe la información. En dado caso que se llegara actualizar alguna tarea y no se mande por ejemplo una descripción, al actualizar esta estará vacía si no se manejan las validaciones necesarias.

Ahora supongamos que queremos actualizar la tarea creada en el punto de POST, dicha actualización contiene la siguiente información: id: f92d8b67-5c4f-4e34-8a86-fadea57a5892, title: Servicios Web Put, body: Esta tarea ha servido de ejemplo para realizar una actualización PUT 

 Put | localhost:3000/api/mongodb/task/f92d8b67-5c4f-4e34-8a86-fadea57a5892


Resultado |

Eliminar proyecto

Las solicitudes DELETE eliminan un recurso de la base de datos de manera permanente.

Ahora supongamos que queremos ELIMINAR la tarea anteriormente actualizada, solo necesitaríamos conocer el ID f92d8b67-5c4f-4e34-8a86-fadea57a5892 

 Delete | localhost:3000/api/mongodb/task/f92d8b67-5c4f-4e34-8a86-fadea57a5892
